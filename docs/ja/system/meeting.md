# {{ $frontmatter.title }}

毎月の例会で議題（提案）を出すシステム。表示名は「唐門月例会議」で、内部データ上は
`Meet_*`（スイッチ系）および `Meeting_Option_*`（条件・スイッチ系）の接頭辞で管理されている。

各 `Meeting_Option_XXX` が例会で提案できる 1 つの議題にあたり、門派規模・名声・各種鍛錬値・
時期などの条件でゲートされる。多くの議題は一度提案すると自己フラグ
（`Meeting_Option_XXX == 0`）によって再提案できなくなる。

## 会議がロックされる月

全 `Meeting_Option` の時期条件は「3年8月上旬」「3年12月上旬」の 2 つのみで、他の月への除外は
存在しない。

- **3年8月上旬**: 章 6-1「四師兄道別」（`MissionCheckData` M5001 系）。この月だけ通常提案が
  全ロックされ、`Meeting_Option_501_01`（四師兄道別専用議題）に差し替わる排他構造。
- 1年10月の「壽宴準備」（南宮家宴）は `M2000` / `M2003` 系という別のミッション ID 体系で
  実装されており、月例会議のロジックには関与しない。外部勢力絡みのイベントは月例会議とは
  独立したシステムとして扱われている。
