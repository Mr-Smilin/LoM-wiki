# 唐門月例会議

::: info ドラフト段階
本ページはデータ採取途中のドラフトです。提案の正確な名称・効果については進行状況で補完予定です。
発動条件・内部IDのデータは確度が高いため、以下の表をそのまま記載しています。提案の実装効果（採択でゲーム内に何が起きるか）については、今後の検証で記入します。
:::

## 概要

唐門月例会議は、月例で開催される唐門の会議に関するシステムです。特定の条件を満たすと、会議の議題として提案ができます。

- **内部prefix**: `Meeting_Option_*` (提案ID), `Meet_*` (会議関連)
- **表示名**: 唐門月例会議 (`Story/N_Meet_01_005`「唐門月例会議」)
- 一度提案すると、フラグにより**再提案不可**になるものが多い
- **ステータス名の対応**: 門派規模（`faction`）／名声（`fame`）／総鍛造（`total-weapon-point`）／
  総煉丹（`total-poison-point`）／医術（技能2101）／道徳（`karma`）／想い人（`lover`）
- **「3年8月上旬を除く」ゲートの正体**: 3年8月上旬は「四師兄道別」(四師兄との別れ)という一回限りの
  メインストーリーイベントの発生時期(`MissionCheckData` M5001系、章6-1で確認)。この月だけ通常の
  提案が全てロックされ、代わりに`Meeting_Option_501_01`(「3年8月上旬のみ」発動)という四師兄の
  遠行に関する専用議題に差し替わる、という排他構造。名声・門派規模等の条件はこの特別月以外なら
  いつでも判定される

## 提案採択メカニズム

- 各提案には独自のフラグ（例: `Meeting_Option_101_01`）が設定され、採択すると1度限りの施行に固定される
- 採択後のストーリー分岐は、`Switch/Meet_Option_*_*.lua` で管理される (詳細は検証中)
- 提案によっては貢献度・資産の消費量に応じてさらに細かい分岐を持つケースもある（例: `Meeting_Option_01_20_*` の分岐）

## 名声(名聲)の稼ぎ方 (待検証：外部攻略情報からの参考情報)

唐門ルートでは名声の安定した入手手段が乏しいとされる。外部攻略サイトの情報によると:

- 外堡を買い戻した後、管家として働くと低確率で名声が増える(ゲーム内データ未裏取り)
- 薪拾い/水汲みの「路人俠」イベントは名声+1(勝利時)がゲーム内データで確認済み
  (`docs/ja/system/training/index.md`「路人俠」の項目、既存記載)

上記1点目(外堡管家)は外部攻略情報のみでゲーム内データの裏取りはできていない。

## 会議がロックされる月

全 `Meeting_Option` の時期条件は「3年8月上旬」「3年12月上旬」の2つのみで、他の月への除外は存在しない。

- **3年8月上旬**: 章6-1「四師兄道別」(`MissionCheckData` M5001系)。この月だけ通常提案が
  全ロックされ、`Meeting_Option_501_01`(四師兄道別専用議題)に差し替わる排他構造(上記参照)。
- 1年10月の「壽宴準備」(南宮家宴)は `M2000` / `M2003` 系という別のミッションID体系で
  実装されており、月例会議のロジックには関与しない。外部勢力絡みのイベントは月例会議とは
  独立したシステムとして扱われている。

## 注記

- **提案名と効果**は、Story key の記述から部分的に推測していますが、完全な確度はまだありません
- **発動条件**は Lua Condition ファイルより直接抽出した値であり、信頼度は高いです
- 時期制約（「〜を除く」「〜のみ」など）は、`ext.GetTimeValue()` で定義されています
  - `ext.GetCurrentTime() ~= ext.GetTimeValue(3, 8, 1)` ＝ 3年8月上旬を**除く**
  - `ext.GetCurrentTime() == ext.GetTimeValue(3, 8, 1)` ＝ 3年8月上旬**のみ**
  - `ext.GetCurrentTime() >= ext.GetTimeValue(3, 12, 1)` ＝ 3年12月上旬**以降**
